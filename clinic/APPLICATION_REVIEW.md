## ⚠️ المشاكل والأخطاء

### 🔴 **مشاكل أمنية حرجة**

#### 1. **مفاتيح Supabase مكشوفة في الكود**
**الموقع:** `services/db.ts` (السطر 7-8)
```typescript
const SUPABASE_URL = 'https://rwbovdtcnrkslrgdjzth.supabase.co';
const SUPABASE_ANON_KEY = 'eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...';
```
**المشكلة:** المفاتيح مكشوفة في الكود ويمكن لأي شخص الوصول إليها  
**الحل:** استخدام متغيرات البيئة (Environment Variables)

#### 2. **كلمات مرور ثابتة في الكود**
**الموقع:** 
- `pages/Stats.tsx` (السطر 28): `password === '0005'`
- `pages/Expenses.tsx` (السطر 29): `password === '1995'`

**المشكلة:** كلمات المرور مكشوفة ويمكن لأي شخص رؤيتها  
**الحل:** استخدام نظام مصادقة حقيقي أو على الأقل نقلها لمتغيرات البيئة

#### 3. **Row Level Security (RLS) مفتوح بالكامل**
**الموقع:** `supabase_schema.sql` (السطر 32-33)
```sql
create policy "Enable all access for all users" on public.patients
for all using (true) with check (true);
```
**المشكلة:** جميع المستخدمين يمكنهم الوصول لجميع البيانات  
**الحل:** تطبيق سياسات أمنية مناسبة حسب الأدوار

#### 4. **عدم وجود التحقق من صحة المدخلات**
- لا يوجد sanitization للمدخلات
- لا يوجد validation للبيانات قبل الحفظ
- خطر SQL Injection (رغم استخدام Supabase الذي يحمي جزئياً)

### 🟡 **مشاكل متوسطة**

#### 1. **إدارة التواريخ**
- استخدام `Date.now()` لتوليد IDs قد يسبب تعارضات
- عدم التعامل مع المناطق الزمنية بشكل صحيح في بعض الأماكن

#### 2. **معالجة الأخطاء**
- بعض الدوال لا تتعامل مع الأخطاء بشكل كامل
- رسائل خطأ غير واضحة للمستخدم في بعض الحالات

#### 3. **الكود المكرر**
- منطق التواريخ مكرر في عدة ملفات
- بعض الدوال يمكن دمجها

#### 4. **الذاكرة المحلية (LocalStorage)**
- لا يوجد حد أقصى لحجم البيانات
- قد تسبب مشاكل في الأداء مع البيانات الكبيرة
