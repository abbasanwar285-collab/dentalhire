# دليل تطبيق الموبايل (Android APK)

تم إعداد مشروع الأندرويد بنجاح وتجهيزه للتحديثات المستقبلية.

## 🔑 مفتاح التوقيع (Keystore)
تم إنشاء مفتاح توقيع خاص بتطبيقك لضمان الأمان وإمكانية التحديث.
- **المسار:** `my-release-key.keystore` (في المجلد الرئيسي للمشروع)
- **كلمة المرور:** `123456`
- **الاسم المستعار (Alias):** `my-key-alias`

> [!IMPORTANT]
> **هام جداً:** احتفظ بهذا الملف وكلمة المرور في مكان آمن. إذا فقدت هذا الملف، لن تتمكن من تحديث التطبيق على هواتف المستخدمين وستضطر لحذف التطبيق وتثبيت نسخة جديدة كلياً.

## 📱 كيفية بناء التطبيق (APK)

بما أنك تستخدم Windows، الطريقة الأفضل والأكثر استقراراً هي استخدام **Android Studio**:

1. افتح مجلد `android` الموجود داخل المشروع باستخدام Android Studio.
   - أو قم بتشغيل الأمر: `npx cap open android`
2. انتظر حتى تنتهي عملية المزامنة (Sync).
3. من القائمة العلوية، اختر **Build** > **Build Bundle(s) / APK(s)** > **Build APK(s)**.
4. سيقوم Android Studio ببناء التطبيق. عند الانتهاء، سيظهر إشعار في الأسفل. اضغط على **locate** للذهاب لمجلد التطبيق.
   - المسار عادة يكون: `android/app/build/outputs/apk/release/app-release.apk`

> [!WARNING]
> **مشكلة محتملة:** إذا ظهر لك خطأ "Your project path contains non-ASCII characters"، فهذا يعني أن اسم المجلد يحتوي على أحرف عربية (مثل `اااا`).
> **الحل:** يجب عليك تغيير اسم المجلد الرئيسي للمشروع إلى الإنجليزية (مثلاً `my-dental-clinic`) ثم إعادة فتح المشروع.

> [!NOTE]
> لقد قمنا بضبط إعدادات `build.gradle` ليقوم بتوقيع التطبيق تلقائياً باستخدام المفتاح الذي أنشأناه عند بناء نسخة `release`.

## 🔄 كيفية تحديث التطبيق مستقبلاً

عندما تقوم بإجراء تعديلات على الكود (مثل تغيير الألوان، إضافة ميزات):

1. قم بتعديل الكود كما تريد.
2. قم ببناء ملفات الويب الجديدة:
   ```bash
   npm run build
   ```
3. قم بمزامنة التعديلات مع مشروع الأندرويد:
   ```bash
   npx cap sync
   ```
4. افتح Android Studio وقم ببناء APK جديد كما في الخطوات السابقة.
5. أرسل ملف APK الجديد للمستخدمين، وسيتم تحديث النسخة القديمة لديهم دون مشاكل (بشرط عدم تغيير ملف الـ keystore).
